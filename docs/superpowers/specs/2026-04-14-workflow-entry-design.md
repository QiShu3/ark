# 专注工作流入口组件重构设计规格 (Spec)

## 1. 目标与背景

当前系统的“工作流快捷入口”区域（在未运行状态下表现为文字按钮，在运行状态下由 `WorkflowProgressBar` 完全接管）在交互层级和视觉表现上不够直观。
本次重构的目标是：
1.  **提升入口卡片 (PlaceholderCard 中的入口块) 的交互效率**：从“全局点击弹窗”改为“左右分栏的一键快切”模式。
2.  **优化进度条组件 (WorkflowProgressBar)**：在保留全区域接管的前提下，重构为“节点步进器 (Node Stepper)”风格，以更清晰、现代的视觉语言表达当前工作流所处的阶段和时间进度。

## 2. 架构与组件改动

### 2.1 入口卡片区域重构 (PlaceholderCard.tsx)

**目标代码位置：** [PlaceholderCard.tsx](file:///Users/qishu/Project/ark/frontend/src/components/PlaceholderCard.tsx) 中处理 `index === 3 && subIndex === 0/1/2/3`（或相关 `split` 逻辑）返回 `WorkflowProgressBar` 及其外层容器的区域。

**设计方案：一键快切卡片 (Quick Action Card)**

*   **静止状态 (未运行工作流)**：
    *   容器划分为两个交互热区（例如使用 `flex` 布局，左侧占 70%~80%，右侧占 20%~30%）。
    *   **左侧热区 (主操作)**：
        *   展示内容：明显的播放图标 (▶) + 默认工作流名称（如 `运行 test工作流`）。
        *   交互：点击直接触发该工作流的启动逻辑（可能需要与后端的 `start_focus` 或启动特定工作流的 API 对接，视现有逻辑而定。若当前未实现直接通过工作流 ID 启动，需梳理 `_handleFocusToggle` 等逻辑以适配“一键启动默认工作流”的需求）。
    *   **右侧热区 (次操作)**：
        *   展示内容：选项/菜单图标 (⋮) 或设置图标 (⚙)。
        *   交互：点击触发 `ark:open-workflow-modal` 事件，打开现有的工作流管理大弹窗。
    *   **视觉样式**：卡片背景采用半透明毛玻璃质感，左右区域通过轻微的底色差异或细微的分割线区分，悬停时分别有高亮反馈。

*   **运行状态 (工作流进行中)**：
    *   保持现有逻辑：如果 `focusWorkflow?.state !== 'normal'` 且有效，整个卡片区域**完全被 `WorkflowProgressBar` 替换**，隐藏上述的“启动/选项”分栏。

### 2.2 进度条组件重构 (WorkflowProgressBar.tsx)

**目标代码位置：** [WorkflowProgressBar.tsx](file:///Users/qishu/Project/ark/frontend/src/components/WorkflowProgressBar.tsx)

**设计方案：节点步进器 (Node Stepper)**

*   **核心布局**：
    *   **顶部信息区**：左侧显示当前阶段名称及剩余时间（如：`专注阶段 1 (24:12)`），右侧显示总进度百分比（可选，或仅保留倒计时以防信息过载）。文字采用更清晰、醒目的字体（如粗体或品牌色强调）。
    *   **主体进度区**：采用“节点与连线”结合的水平时间线。

*   **视觉与动效细节**：
    *   **背景线**：一条贯穿前后的深色半透明细线，代表整个工作流的总长度。
    *   **进度填充线**：从左至右跟随 `metrics.percent` 增长的亮色（蓝色/橙色）高亮线条。
    *   **阶段节点 (Nodes)**：
        *   每个 `phase` 对应时间线上的一个圆点。
        *   **已完成的节点**：小尺寸（例如 12px），填充相应的成功色或品牌色。
        *   **当前的活跃节点 (Active)**：**大尺寸（例如 24px）**，内部可包含 Icon（例如：专注用 🧠/🔥/目标靶心，休息用 ☕/🛋️）。外部添加光晕 (`box-shadow`) 或呼吸动画 (`animate-pulse` 等 Tailwind 类)，以极强地吸引视觉焦点。
        *   **未来的未进行节点**：小尺寸，空心边框，内部深色，表示尚未到达。
    *   **颜色语义**：保留现有的语义，专注阶段（偶数索引）使用蓝色系 (`blue-400`/`blue-500`)，休息阶段（奇数索引）使用橙色系 (`orange-400`/`emerald-400`)。

*   **实现建议**：
    *   重构 `WorkflowProgressBar.tsx` 内部的 DOM 结构，不再单纯使用几个带偏移的 `div`。
    *   可以通过计算当前节点的 `offset` 结合活跃状态，动态切换该节点的 className，注入大尺寸、Icon 和呼吸灯效果。

## 3. 兼容性与边界情况处理

1.  **尺寸自适应**：新的“一键快切卡片”和“节点步进器”需在原有的 Grid 或 Flex 容器（通常为长方形小卡片）内正常渲染，保证小尺寸下不溢出（例如节点过多时，可能需要隐藏过小的未激活节点，或限制最大显示节点数，但这取决于日常工作流的复杂程度，通常 3-5 个阶段不需特殊处理）。
2.  **空状态处理**：如果用户尚未创建任何工作流预设，静止状态的卡片应显示“创建工作流”，点击左侧和右侧都引导至管理弹窗。
3.  **正在进行但无任务 (Edge Case)**：现有的 `workflowForProgress` 逻辑保留，以保证倒计时状态的平滑。
