# 宠物卡片前端复刻页设计

## 概述

本设计为前端新增一个独立的静态复刻页面，用于尽量贴近用户提供的参考截图。

页面目标不是抽象出一套可复用的产品模块，而是在现有 React + TypeScript 项目中，以最低必要工程结构完成一次 `桌面态视觉复刻`。实现优先级明确为：

- 优先还原顶部大圆角搜索舱、粉白色调、卡片排布、柔和投影和整体气质
- 不替换现有 `/apps` 页面
- 不接入真实业务数据
- 不实现真实搜索、筛选、导航行为
- 不要求卡片插画与原图逐像素一致，允许使用近似占位视觉

## 已确认的产品决策

用户在本轮讨论中确认了以下约束：

- 新页面采用 `独立路由`，不覆盖现有应用中心页面
- 实现方式采用 `纯静态拼图式复刻`
- 复刻范围为 `纯视觉 1:1`
- 重点还原页面框架、配色、排版、装饰和卡片结构
- 卡片中的动物插画允许使用近似占位图，不追求原图一致
- 页面只服务于参考截图对应的 `桌面态`
- 页面不需要真实搜索、筛选和响应式重排能力

## 目标

- 在前端项目中新增一个可直接访问的独立演示页
- 让新页面在主观观感上尽量接近参考截图
- 保持对现有页面和路由的侵入最小
- 用尽量少的结构实现可稳定渲染的静态页面

## 非目标

- 不接后端接口
- 不复用现有首页双栏布局
- 不为后续业务扩展建立复杂组件体系
- 不实现搜索输入联动、筛选切换逻辑或工具栏操作
- 不做专门的移动端适配方案
- 不追求参考图中插画资源的一致性

## 当前上下文

当前前端采用 `HashRouter`，受鉴权包裹的页面路由已经包括：

- `/`
- `/apps`
- `/agent-console`
- `/arxiv`

现有 `AppCenter` 是一个独立页面，说明本次复刻页适合以同级独立路由挂载，而不是嵌入首页 `App` 的双栏布局中。

项目全局样式基于 `Tailwind CSS v4` 和 `frontend/src/index.css`。现有大多数页面偏深色或功能型 UI，因此本次复刻页应显式建立自己的浅色背景、阴影和粉色系视觉，不应受现有页面风格牵引。

## 路由与页面归属

### 新增页面

- 新建页面文件：`frontend/src/pages/MockPetMarket.tsx`

### 新增路由

- 在 `frontend/src/main.tsx` 中新增独立受保护路由
- 建议路径：`/mock-pet-market`

这样可以满足以下目标：

- 页面可直接访问和验收
- 不影响现有 `/apps` 使用方式
- 不需要改动首页布局逻辑

## 页面结构

页面整体采用“固定画板感”的桌面结构，自上而下分为三层。

### 1. 外层背景层

- 整体背景为奶白色主底
- 叠加非常淡的粉橘色径向晕染
- 视口高度至少撑满屏幕
- 内容水平居中

### 2. 顶部搜索舱

顶部区域是整个页面的视觉锚点，采用大型白色胶囊容器：

- 超大圆角
- 浅粉色描边
- 柔和扩散阴影
- 轻微内高光

内部结构分两排：

- 第一排：品牌标识、标题标签、少量辅助 pill、右侧工具图标
- 第二排：搜索图标、输入占位文案

该区域只渲染静态视觉，不提供真实输入或点击逻辑。

### 3. 内容区

内容区从上到下包含：

- 小号说明文案，例如“按分类筛选”
- 两个静态筛选 pill
- 一排五张宠物卡片

卡片区域以固定桌面宽度排布为主，不做复杂换行策略优化。

## 卡片结构

每张卡片保持统一尺寸和一致骨架，强调“可爱交易卡”的视觉印象。

### 顶部角标区

- 左上：等级徽标，例如 `Lv.5`
- 右上：状态徽标，例如计数或叶子样式信息
- 徽标使用浅底、小描边、圆角胶囊形态

### 中部插画区

- 占据卡片上半部分的视觉中心
- 使用近似占位图、渐变色块或简单插画容器
- 目标是接近参考图中的体积、位置和留白关系，而不是复刻原始动物素材

### 文本信息区

- 主名称使用更醒目的红粉色
- 右侧小字标识品类或角色标签
- 次级文案使用灰紫色

### 进度条区

- 渲染一条静态粉色进度条
- 只负责视觉存在感，不承载真实数值计算

### 底部数据条

- 使用更浅的分隔背景或顶部边线
- 放置 2 到 3 组小图标加数字
- 数据直接写死，用来还原截图底部信息密度

## 视觉还原策略

### 色彩

主色方向固定为：

- 奶白底色
- 浅粉描边
- 糖果紫点缀
- 薄荷绿辅助状态色
- 灰紫文本次级色

避免复用项目当前偏深色或偏蓝色的默认语气。

### 阴影与光感

- 阴影应偏散、偏软，不使用硬边阴影
- 顶部搜索舱和卡片都应有粉色调柔光
- 卡片可加入非常弱的内发光或顶部高光增强玩具感

### 圆角体系

- 搜索舱圆角最大
- 卡片圆角次之
- pill、角标、底部数据徽标统一走胶囊或软圆角风格

### 字体层级

- 标题与主名称偏粗，但不追求大字号
- 次级文案和统计数字偏小，提升截图中紧凑信息板的感觉
- 整体文案层级要压缩，避免页面看起来比参考图更“现代企业化”

## 插画占位策略

由于用户明确确认插画允许近似占位，本次设计不引入寻找同款动物素材的范围。

允许采用以下任一形式：

- 本地占位图片
- 渐变色块 + emoji / 简化插图
- 柔和背景底托 + 中心图形

约束只有两点：

- 视觉重心必须接近参考图
- 不得破坏卡片整体的可爱、轻盈气质

## 实现策略

### 推荐实现方式

实现采用“单页面 + 少量页面内辅助常量”的轻量结构：

- 主体页面集中在 `MockPetMarket.tsx`
- 卡片数据在文件顶部以内联常量定义
- 使用 Tailwind 原子类完成大部分布局和颜色
- 仅在确有必要时向 `frontend/src/index.css` 补充少量复用 class 或 keyframes

### 不推荐的实现方式

本次不采用以下方向：

- 为纯静态页拆出大量复用组件
- 为卡片数据建立额外状态管理
- 为了追像素而将整页写成大量绝对定位

## 数据与状态

页面不依赖远程数据，也不需要本地状态驱动业务行为。

允许存在的唯一轻量状态是纯展示级状态，例如：

- 如果需要 hover 效果时由 CSS 负责
- 如果需要 map 渲染五张卡片，可使用本地常量数组

不引入：

- `useEffect` 拉取数据
- 搜索关键字状态
- 筛选切换状态

## 错误处理与兼容

由于页面是纯静态复刻页，主要风险不在数据失败，而在样式回退和布局偏差。

应保证：

- 页面在常见桌面宽度下不会报错或出现结构断裂
- 即使占位图未加载，也保留稳定的插画容器尺寸
- 路由接入后不会影响现有鉴权路由行为

## 测试策略

测试只覆盖“页面存在且关键静态结构渲染成功”，不把视觉细节写死到断言中。

建议新增一个轻量页面测试，验证：

- 新页面标题或关键筛选文案存在
- 五张宠物卡片被渲染
- 新路由可以正常挂载页面

同时运行：

- `pnpm check`

如测试文件采用页面级写法，可参照现有 `frontend/src/pages/__tests__/` 风格。

## 验收标准

验收以以下条件为准：

- 新增独立路由页面可正常访问
- 不影响现有 `/apps` 与其他页面
- 顶部搜索舱、筛选区域、五张卡片、粉白色调、软阴影和圆角层次与参考截图保持高度接近
- 页面整体观感明显趋近参考图，而不是现有项目默认 UI 风格
- 插画虽然允许占位，但不能显得突兀或破坏版式重心

## 实施顺序

建议实施顺序如下：

1. 新增页面文件与路由接入
2. 先搭出背景、顶部搜索舱和内容主框架
3. 完成五张卡片的静态结构和占位插画区
4. 微调颜色、阴影、圆角和间距，使观感贴近参考图
5. 新增轻量测试并运行类型检查
