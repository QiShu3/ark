# 开发日志

## 维护规则

- 本文件用于记录每次代码提交后的开发记录。
- 日志按工作分支分组维护，每个分支只保留一个分组标题。
- 每次提交代码后，必须在对应分支分组下新增一条提交记录。
- 同一分支下的新提交放在最上方，旧提交按时间倒序保留。
- 每条记录至少包含：
  - 提交时间
  - 提交哈希
  - 提交信息
  - 用户提出的需求
  - 本次完成的工作
  - 验证情况
- `用户提出的需求` 应优先保留用户原话，只做最小整理。
- 文档提交如果没有额外验证，需明确写明“文档提交，无额外代码验证”。
